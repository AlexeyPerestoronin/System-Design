# Детали

## 1. Функциональные требования (ФТ)
<details>
<summary>click to expand...</summary>

**Функциональные требования = "что нужно сделать?"** - это фиксация требования через ответы на вопросы, которые **можно запрограммировать**:
* в какой среде работает система?
* с данными каких типов работает система?
* каковы сценарии использования?
* что ожидается на входе и выходе?
* как должна вести себя система в тех-то и тех-то случаях?
* как система должна реагировать на различного рода ситуации?
* как система обрабатывает возникающие ошибки?

</details>


## 2. Нефункциональные требования (НФТ)

<details>
<summary>click to expand...</summary>

**Нефункциональные требования (свойства)** - это фиксация требований системы через ответы на вопросы, которые **нельзя прямо запрограммировать**:
* Как должно работать?
    * какова ожидаемая степень доступности?
    * каков необходимый класс надёжности?
    * какова ожидаемая степень отказоустойчивости?
    * какова ожидаемая степень масштабируемости?
    * какой объём аналитики необходимо собирать?
* Какие есть ограничения?
    1. в каких ограничения будет функционировать система?
    * каковы ограничения на время/ресурсы?
    * какое минимальное время отклика?
    * какая предполагается среда функционирования?
* Какие ключевые метрики?
   1. DAU/WAU/MAU = Daily/Weekly/Monthly Active User - количество ожидаемых активных пользователей за определённый период
   * Количество пользователей через 5 лет
   * Количество операций пользователем в день
   * RPS = количество запросов в систему в секунду
   * Объём сетевого трафика за минуту/день/месяц
   * CPU = минимальные/ожидаемые/перспективные вычислительные мощности
   * HDD/SDD = минимальные/ожидаемые/перспективные объёмы файловых хранилищ различных типов
   * HDD/SDD+ = минимальные/ожидаемые/перспективные объёмы увеличения хранимых данных за период (д.м.г.)
   * Какова минимальная и максимальная оценка стоимости системы
*  Какова матрица приоритетности?
![](./priority%20matrix.drawio.png)
    1. **Agility** - способность быстро адаптироваться под постоянно меняющиеся бизнес-требования.
    * **Configurability** - способность быстро адаптировать под решение конкретных зад.
    * **Availability** - качество способности быть доступной (например 99% времени в секундах в год).
    * **Information Security** -  качество способности хранить и передавать информацию в защищённом виде.
    * **Maintainability** -  качество способности быть легко поддерживаемой _на плаву_.
    * **Hight Throughput** - качество способности обрабатывать максимальное количество запросов.
    * **Low Latency** - качество способности по обработки единичного запроса за минимальное время.
    * **Usability** - качество способности толерантности к пользовательскому опыту.
    * **Testability** - качество способности быть легко тестируемой. 
    * **Elasticity** - способность быстро адаптироваться под постоянно меняющуюся нагрузку.

</details>
